{/* ── The Problem ── */}
<section class="landing-section">
  <p class="section-label">The Problem</p>
  <h2 class="section-title">Your CI pipeline is a house of cards.</h2>
  <p class="section-subtitle">
    Every CI system treats pipelines as ephemeral processes. When something goes wrong, you start over.
  </p>

  <div class="pain-grid">
    <div class="pain-card">
      <p class="pain-label">The pain</p>
      <h3>Your Jenkins server crashed at 2am. Again.</h3>
      <p>45-minute build, gone. No state saved. No way to resume. Just start over and hope.</p>
      <p class="fix-label">TemporalCI</p>
      <p>Temporal workflows survive node failures. Your pipeline resumes from the last completed step.</p>
    </div>
    <div class="pain-card">
      <p class="pain-label">The pain</p>
      <h3>500 lines of YAML to build a Go binary.</h3>
      <p>Nested conditionals, matrix strategies, reusable workflows that aren't reusable. YAML was never meant for this.</p>
      <p class="fix-label">TemporalCI</p>
      <p>12 lines of <code>.temporalci.yaml</code>. Flat, readable, no DSL to learn.</p>
    </div>
    <div class="pain-card">
      <p class="pain-label">The pain</p>
      <h3>Debugging means clicking through 47 collapsed log groups.</h3>
      <p>Something failed. Where? Which step? Which line? Good luck finding it in the GitHub Actions log viewer.</p>
      <p class="fix-label">TemporalCI</p>
      <p>Full pod logs on your PR. Presigned S3 links for complete output. Temporal Web UI for workflow inspection.</p>
    </div>
  </div>
</section>

{/* ── Config Showcase ── */}
<section class="landing-section">
  <p class="section-label">Simple by design</p>
  <h2 class="section-title">Your entire CI config. That's it.</h2>
  <p class="section-subtitle">
    No DSL. No plugins. Just declare your steps and let Temporal handle the rest.
  </p>

  <div class="config-showcase">
    <div class="terminal" style="max-width: none;">
      <div class="terminal-header">
        <span class="terminal-dot red"></span>
        <span class="terminal-dot yellow"></span>
        <span class="terminal-dot green"></span>
        <span style="margin-left: 0.5rem; font-size: 0.75rem; color: var(--text-muted);">.temporalci.yaml</span>
      </div>
      <div class="terminal-body">
        <div><span class="muted"># Your entire CI pipeline</span></div>
        <div><span class="cmd">steps:</span></div>
        <div>  <span class="cmd">- name:</span> <span style="color: #a78bfa;">test</span></div>
        <div>    <span class="cmd">image:</span> golang:1.23</div>
        <div>    <span class="cmd">command:</span> go test ./...</div>
        <div>    <span class="cmd">timeout:</span> <span style="color: #fbbf24;">10m</span></div>
        <div>&nbsp;</div>
        <div>  <span class="cmd">- name:</span> <span style="color: #a78bfa;">build</span></div>
        <div>    <span class="cmd">image:</span> golang:1.23</div>
        <div>    <span class="cmd">command:</span> go build -o /artifacts/app ./cmd/server</div>
        <div>    <span class="cmd">depends_on:</span> <span style="color: #34d399;">[test]</span></div>
        <div>&nbsp;</div>
        <div>  <span class="cmd">- name:</span> <span style="color: #a78bfa;">lint</span></div>
        <div>    <span class="cmd">image:</span> golangci/golangci-lint:latest</div>
        <div>    <span class="cmd">command:</span> golangci-lint run</div>
        <div>    <span class="muted"># No depends_on → runs in parallel with test</span></div>
      </div>
    </div>

    <div class="config-annotations">
      <div class="annotation">
        <h4>timeout: 10m</h4>
        <p>Temporal enforces this at the workflow level. No zombie builds. No runaway processes eating your cluster.</p>
      </div>
      <div class="annotation">
        <h4>depends_on: [test]</h4>
        <p>DAG execution. Steps without dependencies run in parallel automatically. Failed deps skip downstream steps.</p>
      </div>
      <div class="annotation">
        <h4>image: golang:1.23</h4>
        <p>Each step runs in its own isolated K8s pod. Proper resource limits, dedicated node pool, automatic cleanup.</p>
      </div>
      <div class="annotation">
        <h4>That's the whole file.</h4>
        <p>No Groovy DSL. No nested YAML matrices. No reusable workflow indirection. Just steps.</p>
      </div>
    </div>
  </div>
</section>

{/* ── Features ── */}
<section class="landing-section">
  <p class="section-label">Features</p>
  <h2 class="section-title">Built for teams that are done babysitting CI.</h2>

  <div class="feature-grid">
    <div class="feature-card">
      <div class="feature-icon">⚡</div>
      <h3>Durable Workflows</h3>
      <p>Powered by Temporal. Your pipeline state survives crashes, restarts, and node evictions. Resume from the last completed step, not from scratch.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">🐳</div>
      <h3>K8s-Native Execution</h3>
      <p>Each CI step runs in its own pod with resource limits and node pool isolation. No shared runners. No noisy neighbors.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">🔄</div>
      <h3>Automatic Retries</h3>
      <p>Per-step retry policies with exponential backoff. Clone fails? Retried 3x. Flaky network? Handled. You configure the policy, Temporal enforces it.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">📊</div>
      <h3>PR-Native Results</h3>
      <p>Commit status, PR comments with per-step timing, collapsible logs, and direct links to the Temporal Web UI. Results where you already work.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">🔐</div>
      <h3>Secure by Default</h3>
      <p>CI pods get zero K8s API access. Network policies block cluster-internal traffic. OIDC federation — no stored credentials. Secrets via AWS Secrets Manager.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">📦</div>
      <h3>One Helm Chart</h3>
      <p>Same chart runs on minikube and EKS Auto Mode. Different values, same architecture. Local dev in minutes, production via GitOps.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">🔀</div>
      <h3>DAG Execution</h3>
      <p>Steps declare dependencies. Independent steps run in parallel. Failed dependencies skip downstream steps automatically. No wasted compute.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">🔍</div>
      <h3>Full Observability</h3>
      <p>Every workflow execution is inspectable and replayable in the Temporal Web UI. See exact inputs, outputs, and timing for every activity.</p>
    </div>
    <div class="feature-card">
      <div class="feature-icon">🚫</div>
      <h3>Cancel &amp; Replace</h3>
      <p>New push on the same branch? Previous workflow is cancelled, cleanup runs, and GitHub shows "cancelled" — not "pending forever."</p>
    </div>
  </div>
</section>
